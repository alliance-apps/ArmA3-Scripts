Tutorial: Marker Finder

Warning

In this tutorial we will change files in your clientside mission, in your @life_server - so please make backups before starting. Seriously, do it. AllianceApps is not responsible for any data loss.

The next steps need to be executed in the clientside mission (.Altis, .Tanoa, .Malden):

Step 1

Dont forget to create backups!

Step 2

Copy the folder markerfinder into /core/.

Open /Functions.hpp or /Functions.h and add the following before the last curly bracket } (after class Vehicle {):

class markerfinder {
	file = "core\markerfinder";
	class animateMarkerFinder {};
	class closeMarkerFinder {};
	class initMarkerFinder {};
	class markerFinder {};
	class markerFinderInput {};
};

Step 3

Edit /stringtable.xml and go to the end of the file. Now add the content of the included stringtable.xml before the last </Project> - tag:.

Step 4

Edit /core/configuration.sqf and add the following before the end of the file:

life_markerFinder = false;

Step 5

Edit /config/Config_Master.hpp and add the following before the end of the file:

#include "..\core\markerfinder\Config_MarkerFinder.hpp"

Step 6

Edit /core/init.sqf and add the following before the end of the file:

call life_fnc_initMarkerFinder;

Congratulations. You have now successfully installed the markerfinder script.

Configuration

The file /core/markerfinder/Config_Markerfinder.hpp contains the configuration for this script.
The license is stored in the file LICENSE.txt. It must not be removed from the product. The script can only be installed on 1 public gameserver at a time (non-public testservers are allowed) - if you want to run the script on several servers you'll need several licenses.
